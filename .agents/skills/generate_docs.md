# スキル: ドキュメントの生成 (Generate Docs)

## 目的

既存のコードベースから、APIドキュメント、ユーザーガイド、開発者向けドキュメントを自動生成し、プロジェクトのドキュメントを充実させます。

## 行動規範

- **保存場所**: 生成したドキュメントは `production_artifacts/docs/` ディレクトリに保存します。
- **言語**: ドキュメントは日本語で記述します。
- **正確性**: コードの実装内容を正確に反映してください。推測で書かないでください。

## 手順

1. **コードの分析**: `app_build/` 内のソースコードを走査し、以下を抽出します。
   - 公開されているAPI エンドポイント
   - 主要な関数・クラスとその役割
   - 設定項目と環境変数
   - データモデル/スキーマ

2. **APIドキュメントの生成** (該当する場合):
   - `production_artifacts/docs/API_Reference.md` に出力します。
   - 各エンドポイントについて以下を記載します：
     - メソッド（GET/POST/PUT/DELETE）
     - パス
     - リクエストパラメータ / ボディ
     - レスポンス形式
     - エラーレスポンス
     - 使用例（curl コマンド）

3. **セットアップガイドの生成**:
   - `production_artifacts/docs/Setup_Guide.md` に出力します。
   - 前提条件、インストール手順、環境変数の設定、初期起動手順を記載します。

4. **アーキテクチャドキュメントの生成**:
   - `production_artifacts/docs/Architecture.md` に出力します。
   - システム構成図（Mermaid記法）、データフロー、主要コンポーネントの責務を記載します。

5. **報告**: 生成したドキュメントの一覧をユーザーに報告します。
