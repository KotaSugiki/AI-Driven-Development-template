# スキル: コードの生成 (Generate Code)

## 目的

フルスタックエンジニアとしての目標は、PMが承認した仕様書に完全に基づき、実際のコードを作成することです。

## 行動規範

- **動的コーディング**: HTML/JSに限定されません。承認された `production_artifacts/Technical_Specification.md` で定義された言語やフレームワークで正確にコードを書く必要があります。
- **保存場所**: 必要なフォルダ構造を正確に維持したまま、すべての生のコードを `app_build/` ディレクトリ内に直接保存します。
- **仕様書厳守**: 推測で進めず、仕様書に書かれていない機能を勝手に追加しないでください。不明点があればユーザーに確認してください。

## 手順

1. **仕様書の確認**: `production_artifacts/Technical_Specification.md` を開き、要件・アーキテクチャ・API設計・DB設計を詳細に分析します。

2. **実装計画の作成**: 
   - 生成するファイルの一覧と依存関係を洗い出します。
   - 実装順序を決定します（依存関係のないファイルを先に作成）。
   - 計画が複雑な場合は、ユーザーに確認を取ります。

3. **基盤の構築**: 
   - プロジェクト設定ファイル（`package.json`、`tsconfig.json`、`requirements.txt` 等）を生成します。
   - ディレクトリ構造を作成します。
   - 環境変数のテンプレート（`.env.example`）を用意します。

4. **コア実装**: 
   - バックエンド → フロントエンドの順序で実装します。
   - 各ファイルにはインラインコメントを日本語で記述します。
   - エラーハンドリングを適切に実装します。

5. **統合**: 
   - コンポーネント間の接続を確認します。
   - API呼び出しの整合性を確認します。

6. **出力確認**: 
   - すべてのファイルが `app_build/` 内に存在することを確認します。
   - `package.json` や `requirements.txt` などの設定ファイルが揃っていることを確認します。
   - コードブロックを省略したり、要約したりしないでください。
