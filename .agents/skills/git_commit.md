# スキル: Git コミット (Git Commit)

## 目的

各開発フェーズの完了時に、適切なコミットメッセージで自動的にGitコミットを行い、変更の追跡性を確保します。

## 行動規範

- **コミットメッセージ**: 日本語で、変更内容が一目で分かるメッセージを作成してください。
- **安全性**: `git push` は**必ずユーザーの確認を取ってから**実行してください。自動プッシュは禁止です。
- **機密情報の保護**: コミット対象に `.env` や秘密情報を含むファイルが含まれていないか必ず確認してください。`.gitignore` に追加すべきファイルがあれば追加してください。

## 手順

1. **変更の確認**: 
   - `git status` で変更されたファイルの一覧を確認します。
   - `git diff --stat` で変更量を把握します。

2. **コミット対象の選定**:
   - 変更内容をレビューし、論理的な単位でコミットをまとめます。
   - 機密情報（`.env`、APIキー等）がステージングに含まれていないか確認します。
   - 必要に応じて `.gitignore` を更新します。

3. **コミットメッセージの生成**:
   - 以下の形式で日本語のコミットメッセージを作成します：
     ```
     [カテゴリ] 変更の要約
     
     - 変更点1の詳細
     - 変更点2の詳細
     ```
   - カテゴリ例: `[feat]` 新機能, `[fix]` バグ修正, `[docs]` ドキュメント, `[refactor]` リファクタリング, `[test]` テスト追加

4. **コミットの実行**:
   - `git add` で対象ファイルをステージング
   - `git commit -m "メッセージ"` でコミット

5. **プッシュの提案**:
   - コミット完了後、ユーザーにプッシュするかどうかを確認します。
   - ユーザーが承認した場合のみ `git push` を実行します。
